# ledgr v0.1.1 Acceptance Tests — Data Ingestion & Snapshotting

These are black-box acceptance tests for v0.1.1. All tests assume a fresh DuckDB unless specified.

---

## AT1 — Snapshot create

**Given** a DB initialized by `ledgr_db_init()`  
**When** `ledgr_snapshot_create(con)` is called  
**Then** a row exists in `snapshots` with `status == CREATED` and `snapshot_id` returned.

---

## AT2 — Import bars CSV (format contract)

**Given** a CREATED snapshot  
**When** `ledgr_snapshot_import_bars_csv()` is called with a bars CSV that matches §6.1  
**Then** rows exist in `snapshot_bars` and values are rounded to 8 decimals.

**And** importing a CSV missing required columns fails with `LEDGR_CSV_FORMAT_ERROR`.

---

## AT3 — Instruments CSV optional / auto-generate

**Given** a bars CSV with instrument_ids {A,B}  
**When** instruments_csv_path is NULL and auto_generate_instruments=TRUE  
**Then** `snapshot_instruments` contains A,B with symbol==instrument_id and defaults.

**When** instruments_csv_path is NULL and auto_generate_instruments=FALSE  
**Then** import fails loud.

---

## AT4 — Seal computes and stores snapshot_hash (bars + instruments)

**Given** a snapshot with imported bars and instruments  
**When** `ledgr_snapshot_seal()` is called  
**Then** snapshot status becomes SEALED, `sealed_at_utc` is not NULL, and `snapshot_hash` is not NULL.

**And** modifying instrument metadata (e.g., multiplier) changes `snapshot_hash`.

---

## AT5 — Seal is atomic

**Given** a snapshot where sealing is forced to fail mid-way (e.g., injected error during hashing)  
**When** `ledgr_snapshot_seal()` errors  
**Then** the snapshot is not SEALED with a partial hash; status is FAILED and error_msg populated.

---

## AT6 — Immutability guard

**Given** a SEALED snapshot  
**When** `ledgr_snapshot_import_bars_csv()` or any snapshot write is attempted  
**Then** it fails loud and the row counts remain unchanged.

---

## AT7 — Tamper detection on load

**Given** a SEALED snapshot  
**And** an external SQL mutation modifies one bar row  
**When** `ledgr_backtest_run()` is called referencing this snapshot  
**Then** it fails loud with `LEDGR_SNAPSHOT_CORRUPTED`.

---

## AT8 — Subset universe allowed

**Given** a SEALED snapshot with instruments {A,B,C}  
**When** config universe is {A,C}  
**Then** backtest setup passes (no requirement to match full snapshot).

---

## AT9 — Per-instrument coverage validation

**Given** a SEALED snapshot where:
- A has bars for 2020–2022
- B has bars for 2021–2023

**When** backtest requests universe {A,B} and range 2020–2023  
**Then** it fails loud with `LEDGR_SNAPSHOT_COVERAGE_ERROR` (because A is missing 2023 and B is missing 2020).

---

## AT10 — Snapshot discovery APIs

**Given** multiple snapshots in different statuses  
**When** `ledgr_snapshot_list(con)` is called  
**Then** it returns rows with columns at least: snapshot_id, status, created_at_utc, instrument_count, bar_count.

**When** `ledgr_snapshot_info(con, snapshot_id)` is called  
**Then** it returns the specified columns and counts.


### AT11 — Empty snapshot seal fails

Given: a snapshot_id with zero rows in snapshot_bars
When: ledgr_snapshot_seal(con, snapshot_id) is called
Then:
- it fails with error code LEDGR_SNAPSHOT_EMPTY
- snapshot.status is not SEALED
- snapshot_hash remains NULL


### AT12 — UTF-8 BOM tolerated

Given: bars CSV encoded as UTF-8 with BOM
When: ledgr_snapshot_import_bars_csv(...)
Then: import succeeds and instrument_id values are correct
