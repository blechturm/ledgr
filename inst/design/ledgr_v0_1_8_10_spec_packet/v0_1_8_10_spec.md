# ledgr v0.1.8.10 Spec

**Status:** Active implementation packet after Batch 0 ticket-cut alignment.
**Target Branch:** `v0.1.8.10`.
**Scope:** Closing pass of the v0.1.8.x single-core arc. Land the
v0.1.8.10 substrate work from the Round-3 spike synthesis: ephemeral sweep
subphase telemetry, matrix-canonical fold substrate and strategy accessors,
event-preserving fold-owned FIFO accounting, yyjsonr options hoisting, and
measurement closeout. In light of the accepted B2 RFC, this packet also scopes
the compiled hot-frame B2 measurement gate: Sub-A in `ledgrcore-spike`, Sub-B in
ledgr `dev/bench/`, and no public compiled execution path unless a later
v0.1.9.x promotion ticket is cut.
**Non-scope for this pass:** public compiled-core promotion or default compiled
execution, durable compiled integration, public `ledgrcore` API/build
authorization, target risk, walk-forward execution, public cost/liquidity APIs,
OMS or paper/live semantics, strategy-authoring helper Pass 2, public benchmark
claims, event-log elision, pure inline aggregate replacement of the event
stream, durable schema redesign beyond explicitly approved typed accounting
facts, and any implementation ticket not cut from this packet after review.

## 1. Thesis

v0.1.8.9 removed the large pure-R buffer-write and per-pulse loop debt. The
v0.1.8.10 spike round showed that the remaining single-core work is smaller
and more architectural:

1. the ephemeral sweep path lacks subphase telemetry, so results/reconstruction
   cost cannot be attributed cleanly;
2. the fold substrate still uses named/vector/list shapes where the accepted
   accessor RFC needs integer-indexed, universe-aligned primitives;
3. FIFO lot accounting currently lives in reconstruction, even though it is a
   fill-caused accounting state transition and belongs in the fold substrate;
4. yyjsonr read/write helpers still construct fixed option objects per call;
5. the K1 verdict and B2 RFC make it cheaper and more direct to measure the
   compiled per-pulse fill-batch hot frame before building a redundant R-only
   attribution harness.

The release should make the fold core a cleaner production-R substrate and run
a bounded compiled hot-frame measurement gate while preserving ledgr's
event-sourced trust boundary. Events remain canonical evidence. Fold-produced
accounting facts are typed derived outputs and parity targets, not replacements
for the event stream. Compiled code must earn promotion through production
parity and wall-recovery gates; v0.1.8.10 does not promote a public compiled
execution path by default.

## 2. Release Goals

v0.1.8.10 has eight release goals:

1. Expose ephemeral sweep subphase telemetry (`t_engine`, `t_results`,
   `t_fills_extract`) in the workload-grid harness.
2. Implement the accepted strategy callback contract addendum: `ctx$vec`,
   `ctx$idx()`, 1-based indexing, error-by-default unknown-id handling with
   `missing = "na"` opt-in, and `ctx$vec$feature(feature_id)`.
3. Convert the fold substrate to matrix-canonical / integer-indexed shapes
   where accepted: `id_to_idx`, primitive `state$positions`, and
   matrix-backed next-bar lookup that preserves fill-model context.
4. Move FIFO lot accounting into the fold core as event-preserving accounting
   ownership: cash, positions, and lot state update as one fill-caused state
   transition.
5. Keep the event log canonical. Inline accounting facts accelerate fresh
   ephemeral sweeps and provide parity evidence; reconstruction remains
   verifier/fallback/readback.
6. Hoist fixed yyjsonr read/write options out of per-call helper bodies.
7. Run the compiled hot-frame B2 measurement gate: Sub-A language/feasibility in
   `ledgrcore-spike`, Sub-B production Pattern B gate in ledgr, with pass /
   review-band / fail disposition.
8. Re-measure the workload grid and peer benchmark using v0.1.8.9's same-host,
   current-source, non-public-claim framing.

The release succeeds when the strategy accessor contract lands, event-stream
and accounting parity hold, ephemeral subphase attribution exists, the
post-v0.1.8.10 production-R baseline and B2 gate outcome are documented, and no
future metric surface is forced to depend on fixed inline aggregates instead of
events.

The B2 gate succeeds only if Pattern B passes parity and clears the RFC's
production wall threshold. If it fails or lands in the review band without a
clear promotion decision, the ephemeral wall attribution spike remains the
fallback diagnostic path for v0.1.9 direction.

## 3. Binding Architecture Decisions

### 3.1 Events Remain Canonical

`ledger_events` and in-memory event rows remain the canonical evidence for a
run. v0.1.8.10 may add typed accounting facts to output handlers, but it must
not remove `ledgr_results(bt, "events")` or make fresh-run metrics depend on
data that cannot be reconstructed from persisted events.

### 3.2 Fold-Owned FIFO Accounting

The fold core owns the fill-caused state transition:

```text
fill intent -> resolved fill -> cash, positions, lot_state, accounting facts
```

This is a substrate decision, not a wall-recovery headline. Spike 12 showed
that moving FIFO work from reconstruction to fold time saves only the JSON
parse / dispatcher slice. The justification is that the R fold should own the
same accounting surface future compiled hot-frame work would measure.

### 3.3 Reconstruction Remains A Verifier / Fallback

`ledgr_sweep_summary_from_ordered_events()` and durable readback helpers remain
available. Fresh ephemeral sweeps may use inline facts instead of replaying the
event stream, but reconstruction must still prove parity and support durable
readback.

### 3.4 Fill-Model Contract Preservation

The next-bar optimization is not a `next_bar -> next_open_price` swap.
`ledgr_next_open_fill_proposal()` currently consumes `instrument_id`, `ts_utc`,
`open`, optional OHLCV, final-bar `NO_FILL` behavior, and a cost-resolver
`fill_context`. The matrix-backed implementation must preserve those
observable fields.

### 3.5 Public Context Shape

Public `ctx` remains a list with fresh public slots per pulse. Internal reusable
environments may be used behind helper construction, but public slot retention
and mutation semantics stay documented-convention read-only. `ctx$positions`
keeps the accepted sparse named shape; `ctx$vec$positions` is the
universe-aligned primitive view.

### 3.6 Compiled Hot Frame B2 Is A Measurement Gate

The B2 RFC binds a direct measurement path, not compiled promotion. Ticket
`LDG-2522` has two sub-artifacts:

- Sub-A in `ledgrcore-spike`: language/feasibility, small-fixture parity, build
  flags, and cross-platform parity smoke.
- Sub-B in ledgr: production decision-bearing gate through the real fold path
  using an internal, unexported execution-spec enum:
  `compiled_accounting_model = NULL | "spot_fifo"`. `NULL` is the default
  canonical R fold path; `"spot_fifo"` is the only compiled capability
  v0.1.8.10 may measure.

Pattern B is the only decision-bearing design: compiled per-pulse fill-batch
processing with compiled event accumulation and no per-fill R handler writes.
Pattern A may be used as a parity/debug staging shim only. Promotion requires
Pattern B to deliver at least 30s wall recovery on the LDG-2479
`density_high_xlarge_ephemeral` cell and pass all parity gates. A 15-30s
recovery with parity is a review band; less than 15s or any parity failure parks
B2 and makes the ephemeral attribution spike the next diagnostic path.

The first-cut compiled scope is post-resolution fill-batch work: fresh BUY/SELL
FIFO lot-state transition, cash and positions mutation, event row value
construction, and typed event accumulation. R keeps strategy execution, context,
target validation/risk, next-open proposal, cost resolution, features, equity,
metrics, durable persistence, and replay.

This is a spot-asset FIFO fill-batch accelerator, not a general compiled fold
core or a derivatives accounting model. Unsupported
`compiled_accounting_model` values must fail closed with a named error rather
than silently routing through compiled code or falling back after partial
compiled execution. Future derivatives, margin, options, or other accounting
models require separate model values, RFC scope, and parity gates; they must not
be added by mutating the spot-FIFO kernel.

## 4. Workstreams

### 4.1 Ephemeral Sweep Subphase Telemetry

Add `t_engine`, `t_results`, and `t_fills_extract` to sweep telemetry and the
workload-grid CSV. This is attribution infrastructure with no intended direct
wall recovery.

Acceptance gates:

- `density_high_xlarge_ephemeral` records engine/results/fills-extract
  subphases.
- Durable telemetry remains unchanged or explicitly mapped.
- The telemetry addition has negligible wall overhead on smoke and record
  presets.

### 4.2 Matrix-Canonical Fold Substrate And Accessors

Implement the accepted callback addendum and the substrate surfaces it consumes:

- `id_to_idx` map built at execution-spec / fold setup;
- primitive internal `state$positions`;
- `ctx$idx(id, missing = c("error", "na"))`;
- `ctx$vec` namespace for OHLCV, positions, and
  `ctx$vec$feature(feature_id)`;
- matrix-backed next-bar lookup with minimal execution-bar construction.

Acceptance gates:

- Existing scalar helpers (`ctx$close(id)`, `ctx$feature(id, feature_id)`) keep
  their contract.
- Unknown-id behavior matches the accessor RFC.
- Shuffled-order fixtures prove name/index alignment.
- Fill-model tests cover `instrument_id`, `ts_utc`, optional OHLCV, and
  final-bar warning behavior.
- Strategy guide and contracts document the high-throughput path.

### 4.3 Fold-Owned FIFO Accounting And Inline State Capture

Move FIFO lot accounting into the fold state transition and emit typed
accounting facts to output handlers. Fresh ephemeral sweeps may bypass
reconstruction for equity/fills/metrics when parity gates pass.

Acceptance gates:

- Event log preserved: event IDs, order, event sequence, and materialized
  event rows remain compatible unless an explicit versioned identity change is
  approved.
- Equity parity: inline equity matches reconstruction-derived equity within the
  existing named tolerance mechanism.
- Fill parity: OPEN/CLOSE split rows, quantities, prices, fees, action labels,
  and realized PnL match reconstruction.
- Lot-state parity: cumulative `event_realized` and `event_cost_basis` match
  reconstruction.
- Opening-position / CASHFLOW branch parity is covered.
- Invalid side, BUY_TO_COVER while long, SELL_SHORT while short, malformed meta,
  and rejected-fill behavior match durable extraction.
- Durable readback remains reconstruction-compatible.
- Strategy callback cannot observe same-pulse post-fill lot state.

### 4.4 yyjsonr Options Hoist

Hoist fixed `yyjsonr::opts_read_json()` and `opts_write_json()` configuration
out of per-call helper bodies in:

- `ledgr_json_read_nested()`;
- `ledgr_json_read_config()`;
- `ledgr_json_write_canonical_v2()`.

Acceptance gates:

- Canonical JSON v2 byte fixtures stay unchanged.
- Nested/config read-shape tests stay unchanged.
- Record-scale read/reopen paths show directionally lower read overhead.

### 4.5 Compiled Hot Frame B2 Gate

Run the accepted B2-first measurement gate.

Acceptance gates:

- Sub-A artifact exists in `ledgrcore-spike` and records language choice,
  build flags, small-fixture parity, and cross-platform parity smoke.
- Sub-B uses the real ledgr fold path through an internal, disabled-by-default
  `compiled_accounting_model` execution-spec enum; no instrumented-copy or
  `assignInNamespace` gate is accepted.
- `compiled_accounting_model` is closed in v0.1.8.10: `NULL` means the
  canonical R fold path, `"spot_fifo"` means the internal spot-asset FIFO
  fill-batch accelerator, and any other value fails fast with a named
  unsupported-accounting-model error.
- Pattern B is measured on the LDG-2479 `density_high_xlarge_ephemeral`
  production cell.
- Parity covers the Round-3 substrate gates plus B2 fresh/replay side semantics:
  event log, equity, fills, lot state, opening-position/CASHFLOW, invalid and
  semantic-violation cases, durable readback compatibility, no strategy
  lookahead, fresh BUY/SELL, and replay alias preservation.
- Outcome is classified as pass (`>= 30s` recovery and all parity pass), review
  band (`15s <= recovery < 30s` and all parity pass), or fail (`< 15s` or any
  parity failure).
- No public compiled execution path is enabled by this ticket.
- Closeout language must describe the B2 path as a spot-asset FIFO fill-batch
  accelerator or equivalent scoped phrase, not as a general compiled fold core.

### 4.6 Parked Spike Disposition

Record final v0.1.8.10 dispositions for parked spikes:

- split/gsplit reconstruction bucket as fallback only;
- reusable pulse-context env as v0.1.9+ helper-attachment investigation;
- pulse-seed mixer as below threshold;
- alias-map normalization as subsumed by bulk feature reads;
- any remaining small cleanup lanes explicitly deferred or ticketed.

## 5. Measurement Discipline

Each implementation lane needs a before/after attribution row before the next
lane starts, except packet alignment and final release gate. For v0.1.8.10 the
load-bearing measurement surfaces are:

- `density_high_xlarge_ephemeral` subphase telemetry;
- `density_high_xlarge_durable` and `density_high_xlarge_ephemeral` wall;
- peer benchmark durable/ephemeral phase rows;
- event/equity/fill/lot parity fixtures;
- B2 Pattern B LDG-2479 xlarge ephemeral wall recovery and parity outcome;
- accessor helper microbenchmarks only as mechanism evidence, not release
  claims.

Closeout language must keep peer-benchmark ratios and workload-grid xlarge
walls separate. B2 language must stay scoped to the spot-asset FIFO accelerator
surface. No public benchmark claim language is allowed.

---

## 6. Deferred Scope

The following remain deferred:

- public compiled core promotion, default compiled execution, and public
  `ledgrcore` build/API authorization;
- durable compiled integration;
- derivatives, margin, options, or other non-spot-FIFO compiled accounting
  models;
- target risk, walk-forward execution, OMS, paper/live, public cost/liquidity
  APIs, and TCA/reporting;
- strategy-authoring helper Pass 2 extensions;
- eliminating reconstruction code;
- changing event schema or canonical JSON byte format;
- benchmark marketing or hosted speed claims.

---

## 7. Release Gate

Before release closeout:

- ticket markdown, YAML, batch plan, and design README agree;
- targeted tests for each touched surface pass;
- full test suite passes;
- package build and `R CMD check --no-manual --no-build-vignettes` pass or
  caveats are explicitly accepted;
- workload-grid and peer benchmark closeout exists;
- B2 gate disposition is recorded without implying public compiled promotion;
- generated local artifacts are not committed;
- NEWS documents v0.1.8.10 as substrate/accounting and telemetry work, not as
  a public speed-rank release.
