## Ticket 2.5 — Config Validation

**Implements:** Spec §5 validation rules  
**Output:** fail-loud config validation with clear errors

### Steps
1. Validate required fields exist (`db_path`, `engine.seed`, `universe.instrument_ids`, `backtest.start_ts_utc`, `backtest.end_ts_utc`, `backtest.initial_cash`, etc.)
2. Validate types and constraints:
   - `seed` integer
   - non-empty universe
   - `start_ts_utc <= end_ts_utc`

3. Provide actionable error messages (use `rlang::abort()`)

### Files
- `R/utils/config-validate.R`
- `tests/testthat/test-config-validate.R`

### Tests
- empty universe fails
- invalid time range fails
- missing initial_cash fails


## Ticket 3 — Data Hashing (Run-relevant Snapshot Fingerprint)

**Implements:** Spec §7.1 data_hash  
**Output:** deterministic `data_hash` for the run

### Steps
1. Query run-relevant bars subset ordered by `(instrument_id, ts_utc)`
2. Round numeric bar columns to **8 decimal places** before hashing (spec requirement)
3. Hash in a columnar/streaming fashion where possible (avoid huge R objects)
4. Store in `runs.data_hash`

### Files
- `R/utils/data-hash.R`
- `tests/testthat/test-data-hash.R`

### Tests
- identical bars produce identical data_hash
- row order differences do not change hash (ordering enforced)

## Ticket 9 — Backtest Runner (EOD Pulse Loop)

**Implements:** Spec §8, acceptance AT2/AT7  
**Output:** end-to-end runner

1. Create run row, status transitions
2. For each ts in range:
   - build PulseContext
   - compute features
   - call strategy
   - translate targets to deltas
   - simulate fills at next open
   - append ledger events
   - update equity_curve (or compute after loop)
3. Persist status DONE/FAILED with error_msg

### Files
- `R/engine/backtest-runner.R`
- `tests/testthat/test-backtest-integration.R`

### Tests
